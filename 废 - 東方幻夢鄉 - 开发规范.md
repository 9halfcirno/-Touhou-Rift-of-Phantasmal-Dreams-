##关于**TH**中部分对象格式
**TH.ID**表示法
*TH.ID*是为了方便区分实际的文件路径和游戏的资源路径做区分
*TH.ID*用于访问*TH.Resource*对象的资源属性
*TH.ID*用于在*new TH.Class()*创建时作为传入的参数
```javascript
//完整的TH.ID
TH.ID = "PATH:TYPE_A_B_C_…";
//PATH: 资源来源(区分本体及扩展)
//TYPE: 资源类型(区分资源类型)
//A, B, C, …: 详细路径

//资源来源上下文可推断的TH.ID
TH.ID = "TYPE_A_B_C_…"
//在资源类型明确的情况下省略PATH
//A, B, C, …: 详细路径

//所有上下文均可推断的扩展
TH.ID = "A_B_C_…"

//example
//-"th:texture_block_dream_floor"
//-"th:block_dream_floor" // 在图片相关上下文可用
//-"block_dream_floor" //在图片、TH.STORY_PATH上下文可用
```
**TH.Resource.["image"/"audio"/"model"]**表示法
*TH.Resource.["image"/"audio"/"model"]*并非实际使用在场景中的资源对象，而是定义资源的对象
*TH.Resource.image*会被*TH.Tecture*类使用
*TH.Resource.audio*会被*TH.Audio*类使用
*TH.Resource.model*会被*TH.Model*类使用
其中的url路径会依据type自动推断
| type   | baseurl                   | url                         |
|--------|---------------------------|-----------------------------|
| image  | "resources/th/textures/"  | "blocks/dream/floor.png"    |
| audio  | "resources/th/audios/"    | "entities/player/eat.ogg"   |
| model  | "resources/th/models/"    | "buildings/homo/house.json" |
| define | "resources/th/defines/"   | "defines/textures.js"       |
```javascript
// TH.Resource.image
let image = {
	type: "image",
	url: "blocks/dream/floor.png",
}
// TH.Resource.audio
let audio = {
	type: "audio",
	url: "entities/player/eat.ogg"
}
// TH.Resource.model
let model = {
	type: "model",
	url: "buildings/homo/house.json"
}
```
